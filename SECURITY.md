# Security Policy

## Reporting Vulnerabilities

If you find a security vulnerability in Siyabona, please report it responsibly:

**DO NOT open a public issue.**

Instead, email the maintainers or use GitHub's "Report a vulnerability" feature on the Security tab.

Include:
- Description of the vulnerability  
- Steps to reproduce
- Potential impact

We aim to respond within 48 hours.

## Data Privacy
Contributors must not log or store sensitive user data in plaintext. Anonymize test data in PRs.
